## 共用寫作指南

### 品牌語氣（取自 positioning §4）

| 維度 | 規格 |
|------|------|
| 正式度 | 中性偏正式（書面語近報導體） |
| 專業度 | 白話為主，必要時補專業術語 |
| 情感 | 理性為主、誠懇而非浮誇 |
| 人稱 | **「我們」**（協會立場）、**「你」**（讀者，**不用「您」**以降低距離） |

### 共用用語規範

| 使用（推薦） | 避免（禁止） | 原因 |
|------------|-------------|------|
| 跨域、開源、實作、長期、信任 | 賦能、生態圈、全方位解決方案、革命性 | positioning §4 禁用詞 |
| AI 整合、領域知識、夥伴 | AI 革命、AI 浪潮、顛覆 | 避免空泛宣傳語 |
| 全部免費、無會員費 | 「優惠」「特價」「限時」 | 反向定位「不收費」需明確 |
| 公益專案、開源 | 「服務」「方案」「產品」 | 非商業立場 |
| 圍棋 / 圍棋對局 | 「下棋」泛稱 | 品牌字保留 |

### 格式規範

| 項目 | 規格 |
|------|------|
| 段落長度 | 最多 4 行（行動裝置友善） |
| 句子長度 | 最多 50 字 |
| 列表使用 | ≥ 3 項時改用列表 |
| 強調方式 | **粗體**（不用底線、不用紅字、不用閃爍） |
| 全形／半形 | 中文用全形標點；中英混排時英文兩側補半形空格 |
| 數字 | 半形阿拉伯數字（含中文混排） |

### 「無棋會分支」共識（適用所有頁面）

> 協會目前尚無定期實體棋會（2026-05-20 用戶確認）。所有文案：
> - **不暗示**「我們已有定期棋會」「來棋會見面」
> - 「圍棋作為信任場域」可以提，但定位為**長期願景**而非現有證據
> - 「先見面聊聊」CTA 採 mailto 主旨選項，**不創造「棋會邀請」按鈕**

### i18n 翻譯策略（11 語系）

| 語系 | 處理方式 |
|------|---------|
| zh-tw（原始）| 寫繁中為母版 |
| zh-cn / zh-hk | 用簡繁／粵語人工轉寫（必要時 LLM 輔助 + 人工校對）|
| en | **L1 主敘事**（首頁／合作邀請／實績卡）獨立撰寫，**非機翻**；其餘採機翻 + 抽查 |
| ja / ko | 機翻 + 抽查（特別注意敬語等級）|
| es / pt / hi / id / ar | 機翻；阿拉伯文需 RTL 排版檢查 |

# §A — 首頁規格（`src/pages/index.js` 大改）

## 基本資訊

| 項目 | 內容 |
|------|------|
| 頁面 URL | `/`（11 語系 root，如 `/en/`、`/ja/`）|
| 頁面類型 | 首頁 / Landing |

| 優先級 | P0a + P0b |
| 來源 | strategy.md §4.1 |

## 1. 頁面目標

**主要目標**：讓 Persona A / B 在 5 秒內判斷「值得花時間了解」、30 秒內點 CTA。

**次要目標**：
1. 建立 SEO 對「中小企業 AI 導入」「開源 AI 工具」等長尾關鍵字的對應
2. 讓 Persona C（工程師）找到 GitHub 入口
3. 讓 Persona D（圍棋棋友）能跳到 `/docs/learn/` 不被擋

**成功指標**：

| 指標 | 目標值 | 測量方式 |
|------|--------|----------|
| About 頁停留時間中位數（從首頁點進）| ≥ 60s（Phase 1 後）/ ≥ 90s（Phase 3 後）| Umami |
| mailto 點擊事件 | ≥ 0（Phase 1 30 天內出現非零）| Umami |
| 首頁→會員實績頁的點擊率 | ≥ 5% | Umami |

## 2. 目標受眾

| 順序 | 受眾 | 來這頁的目的 | 進入方式 | 下一步期望 |
|------|------|-----------|---------|----------|
| 1 | Persona A（二代接班型）| 判斷協會是否值得了解 | Google 搜尋「中小企業 AI 導入」/ 朋友轉貼連結 | 點 mailto 寫信 / 看 11 位會員 |
| 2 | Persona B（退休專業型）| 看是否有同年齡層、同產業同儕 | 圍棋圈耳語 / 朋友推薦 | 點「先見面聊聊」mailto |
| 3 | Persona C（AI 工程師）| 看是否有真實使用者／領域知識的開源專案 | GitHub trending / X | 點到 GitHub repos |
| 4 | Persona D（圍棋棋友）| 找學習資源 | Google「圍棋入門」/「AlphaGo」| 跳到 `/docs/learn/` |

## 3. 關鍵訊息

| 順序 | 訊息 | 呈現方式 |
|------|------|---------|
| 1 | **11 位跨域夥伴 × 41+ 開源 AI 工具 × 全部免費** | Hero stats（半屏視覺） |

## 設計 token 一致性檢查（**新增 2026-05-21**）

> **本檢查項是 2026-05-21 補列**：USP 區塊曾因 hardcode hex / rem 而違反專案 design system，2026-05-21 已建立 `src/css/design-tokens.css`（oklch 色票 + 統一 type scale + spacing/radius/shadow），所有新組件 CSS 必須對齊。

- [ ] **顏色**：不引入新的 hex literal；改用 semantic token（`var(--color-brand)`、`var(--color-text-primary)`、`var(--color-accent-warm)` 等）。所有顏色來源 = `design-tokens.css` 的 oklch primitives
- [ ] **字體大小**：不引入新的 rem literal；只使用 `var(--font-size-xs/sm/base/md/lg/xl/2xl/3xl/4xl)`。如缺尺寸先在 design-tokens.css 加 token

- [ ] **圓角**：只使用 `var(--radius-sm/md/lg/xl/pill)`

- [ ] **避免 hardcode dark-mode 顏色**：使用 semantic token（深色模式由 design-tokens.css 統一處理），不寫 `[data-theme='dark'] .foo { color: #bbb }`
