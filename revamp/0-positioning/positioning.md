# 品牌定位文件

> **改版方向**：路徑 B — 三層並存
> - L1（主敘事）：商界夥伴網絡，以圍棋為紐帶，共同應用 AI
> - L2（支撐）：開源研究與 AI 工具，作為「夥伴一起做出來的成果」
> - L3（延伸）：圍棋教育與文化，作為對社會的延伸服務
>
> **本文件目的**：作為後續 `1-discovery` 到 `5-content-spec` 所有階段的判斷準據；首頁文案、navbar、CTA、會員頁、SEO 全部對齊本文件。

## 1. 核心資訊

| 項目 | 內容 |
|------|------|
| **網站名稱** | 台灣好棋寶寶協會（weiqi.kids） |
| **一句話描述** | 由理事長作為 AI 整合者，串接 11 位跨域夥伴的領域專業，以圍棋為紐帶，共同產出開源 AI 工具與圍棋文化資源 |
| **核心價值主張** | 把圍棋的「對局思維」帶進商業協作：信任、長期、開源、跨域。**領域專家提供問題與知識、理事長負責 AI 工程整合**，把產業實務變成可被社會使用的開源工具 |
| **夥伴端收益**（為什麼參加？）| (1) **本業 AI 賦能**：自家領域的問題被做成 AI 工具，可直接用於本業；(2) **跨域人脈**：律師＋ISO＋技術＋行銷＋醫療同桌長期合作；(3) **公益／ESG 形象**：以開源公益專案掛名共創者；(4) **AI 世代低風險入場券**：不需要自建技術團隊就能參與 AI 落地 |

## 2. 目標受眾

### 主要受眾（Primary）— 商界合作夥伴

| 項目 | 描述 |
|------|------|
| **是誰** | 30–55 歲的中小企業主、專業人士（律師、會計師、顧問、創業者、產業專家）。已經在各自領域有實績，正在尋找「跨域信任夥伴」共同探索 AI 落地 |
| **目標／需求** | (1) 找到能長期合作、有共同價值觀的跨域夥伴；(2) 在自己產業導入 AI，但需要懂技術的同伴；(3) 把自己的專業知識變成可被 AI 使用的公益資產 |
| **痛點** | (1) 商業社團多半是純人脈交換，沒有共同產出；(2) AI 應用案例多但「自己怎麼做」沒人帶；(3) 沒有低風險、能長期觀察彼此誠信的場域 |
| **使用場景** | 想加入協會前的盡職調查；想看「這群人到底一起做了什麼」；想找特定產業的合作對象；活動報名與會員實績查詢 |

#### 主要受眾的具體 Persona（草稿，待 `3-analysis` 驗證）

> **[假設]** 將寬範圍 ICP 收斂為 2 個具體 persona，便於後續內容判斷「該說給誰聽」。

| Persona | A — 二代接班型 | B — 退休專業型 |
|---------|---------------|---------------|

| **身分** | 中小企業二代或新任主管 | 律師／會計師／醫師／顧問退居二線 |
| **動機** | 想用 AI 強化既有服務、為下一代企業準備轉型 | 想把累積一輩子的領域知識變成公益資產 |
| **痛點** | 沒有內部技術團隊、不知道從哪開始 | 個人能力強但難找對等的跨域合作對象 |
| **可付出** | 商業案例、產業 know-how、預算 | 領域知識、信譽背書、社會連結 |

### 次要受眾（Secondary）— AI／開源／圍棋愛好者

| 項目 | 描述 |
|------|------|
| **是誰** | (a) AI 工程師、研究者；(b) 圍棋棋友、教育工作者、家長 |
| **目標／需求** | (a) 找到值得貢獻的開源專案、看實戰應用案例；(b) 學習圍棋資源、了解 AlphaGo 演進、給孩子的圍棋啟蒙 |
| **痛點** | (a) 多數開源 AI 專案缺領域知識／真實使用者；(b) 圍棋教學資源散亂、AI 圍棋的故事沒人系統整理 |
| **使用場景** | (a) GitHub 連結進來看專案、找協作機會；(b) 從搜尋引擎進來找學習資源 |

> **[假設]** 主要受眾的具體痛點是依現有 11 位創始會員的跨產業組成（技術、生技 ISO、律師、行銷、策略、整復…）反推，需要在 `3-analysis` 階段用實際訪談或問卷驗證。

## 3. 差異化定位

| 項目 | 內容 |
|------|------|
| **與競品的差異** | (1) 不是純扶輪／獅子會式的社交網絡 — 我們有共同「開源產出」當證據；(2) 不是純 AI 社群 — 我們有圍棋對局與長期人際信任當基礎；(3) 不是純圍棋協會 — 我們把圍棋當紐帶不當目的 |
| **獨特優勢** | (1) **40 個公益專案全部開源**（agent.writer / six-hats / tax-ai / huatuo-ai / tapwater-ai / bless.link / tma / social-listening / 等等）— 每個都對應協會夥伴的本業領域；(2) **「AI 整合者 × 領域專家」分工**：理事長負責工程整合、夥伴貢獻產業知識，比純技術社群更落地、比純商業社團更有產出；(3) **圍棋對局形成的長期信任**：盤上多次交手，比履歷更能看出一個人（**前提：需要有定期實體棋會**，待 `1-discovery` 盤點 `docs/about/activities/` 確認；若無，須補建立月例會等機制以支撐此論述） |
| **不做什麼** | (1) 不做純社交聯誼或商務媒合平台；(2) 不做封閉付費會員制（核心產出全開源）；(3) **不做付費會員制圍棋培訓班、不收實體課學費、不做升段考試輔導**；圍棋教學內容仍以 AI 工具產出形式公開保留於 L3，作為「AI 技能對外的表現」 |

## 4. 品牌調性

| 項目 | 描述 |
|------|------|
| **語氣** | 專業而不冷漠、誠懇而不浮誇、實作派而不空談。書面語近報導體，不用行銷術語（避免「賦能」「生態圈」「全方位解決方案」這類詞） |
| **視覺風格** | 簡約、留白、棋盤格／棋子的隱喻元素低調呈現；色彩克制（圍棋黑白為主、輔以單一暖色當品牌色）；不用花俏動畫 |
| **關鍵字** | 開源、跨域、信任、實作、長期 |

## 5. 成功指標

KPI 分兩層：**先行指標（Leading）**反映流量與互動，容易達成、用於早期觀察；**結果指標（Hard）**反映真實商業轉換，是改版成敗的最終判準。

| 類型 | 指標 | 目標值（90 天內） | 測量方式 |
|------|------|-------------------|----------|
| 🟡 Leading | 「合作提案」mailto: 點擊數 | ≥ 30 次 | Plausible 事件追蹤 |
| 🟡 Leading | About 頁停留時間中位數 | ≥ 90 秒 | Plausible / GA |
| 🟡 Leading | 會員實績頁 PV 佔比 | 從 0 → ≥ 8% 全站 PV | Plausible |
| 🟡 Leading | AI 工具頁外連點擊（GitHub）| ≥ 100 次 | Plausible 外連事件 |
| 🟡 Leading | 品牌字搜尋「好棋寶寶」+「AI」聯合查詢 impression | 出現於 GSC 報表 | Google Search Console |
| 🔴 **Hard** | **實際 inbound 商務洽談數**（真的寄出的提案信） | **≥ 5 件** | 信箱實際收到的 thread 計數 |
| 🔴 **Hard** | **由 inbound 洽談轉為實際合作的案例數** | **≥ 1 件** | 半年內可追蹤的合作協議 |

> **[假設]** 上述數字為「能感受到改版有效果」的最小門檻，實際基準需在 `1-discovery` 取得改版前 30 天的數據後重新校準。
>
> ⚠️ **判讀規則**：若 Leading 指標達標但 Hard 指標未達標，代表「網站吸引點擊但無法促成真正商務洽談」— 屬於文案／信任度／案例說服力不足，需重新檢視；不可僅以 Leading 指標宣稱改版成功。

## 6. 改版目標

P0 細分為 P0a（敘事骨幹，必須先做）→ P0b（依賴 P0a 才能做的內容填充）；P1/P2 依舊。

| 優先級 | 目標 | 說明 |
|--------|------|------|
| **P0a** | 首頁主敘事重寫為「商界夥伴 × AI × 圍棋」 | 把現有 hero 的「開源研究社群．從圍棋出發」改為以「夥伴」為核心；CTA 改為「合作提案」「認識夥伴」（保留「研究成果」） |
| **P0a** | Meta description 全站更新（移除「圍棋教學」為主軸的舊文案） | 影響 SEO 與社群分享預覽；和 P0a 首頁文案同步改 |
| **P0a** | 新增「合作提案」入口 `mailto:lightman.chang@gmail.com` | 首頁主 CTA 與 navbar 加入「合作提案」連結；不做表單頁 |
| **P0b** | About 頁從「協會推廣圍棋文化」改為「我們是誰、為什麼一起、做了什麼」 | 目前使命／願景／歷程全部寫「待補充」，必須補齊；以實際合作故事為主，不用空泛宣言（依賴 P0a 敘事骨幹） |
| **P0b** | 11 位會員的「實績卡」頁面：每人本業＋協會內貢獻＋對應的 AI 工具 | 把「11 位創始會員」從數字變成可驗證的人；每位會員連結到對應的 AI 專案（如鄭骨館 → huatuo-ai） |
| **P0b** | 首頁加入「合作案例敘事」區塊 | 由理事長主筆，呈現「夥伴提供的領域問題 → 理事長 AI 整合 → 開源產出」的具體故事（3–5 個案例卡片）；依賴 P0b 會員實績卡內容 |
| **P1** | 「三大研究軸線」改為「三大合作面向」並改順序 | 案例 → 工具 → 論文（從「夥伴一起做了什麼」往「公開研究成果」遞進） |
| **P1** | Navbar 結構從「給棋友／給工程師」改為「夥伴／實績／工具／圍棋」 | 反映 L1→L2→L3 三層結構；避免二分法把商界主受眾擋在外面 |
| **P2** | 「圍棋資源」整合為單一入口（學圍棋／AlphaGo／動畫教室／技術文件） | 收進次要受眾路徑，不擠首頁 |
| **P2** | 各會員專業領域的標籤化（產業／能力／合作意向）支援篩選 | 讓主要受眾能依需求找對人 |
| **P2** | 多語系版本同步更新（11 語系） | 主敘事改寫後，i18n 翻譯一次到位 |

## 7. 內容三層映射（路徑 B 結構）

```
┌─────────────────────────────────────────────────────────────┐
│  L1（主）：商界夥伴網絡                                       │
│  ├─ 我們是誰（11 位會員、跨產業）                              │
│  ├─ 一起做了什麼（合作案例、活動實績）                         │
│  └─ 怎麼加入（合作提案、商務聯繫）                             │
├─────────────────────────────────────────────────────────────┤
│  L2（支撐）：開源 AI 與研究成果                                │
│  ├─ AI 工具（40 個公益專案，GitHub 連結）                      │
│  ├─ 產業情報（記憶體／能源／房市 等垂直研究）                   │
│  └─ 學術論文                                                  │
├─────────────────────────────────────────────────────────────┤
│  L3（延伸）：圍棋文化與教育                                    │
│  ├─ 學圍棋資源（次要受眾入口）                                  │
│  ├─ AlphaGo 演進故事                                          │
│  └─ 動畫教室、技術文件                                         │
└─────────────────────────────────────────────────────────────┘
```

每層對應的首頁版面、navbar 分組、SEO 關鍵字策略，將在 `4-strategy` 詳細規劃。
