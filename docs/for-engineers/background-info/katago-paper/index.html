KataGo 論文解讀

KataGo 是由 David Wu 開發的開源圍棋 AI，其論文《Accelerating Self-Play Learning in Go》發表於 2019 年。KataGo 以更少的計算資源達到超越 ELF OpenGo 的棋力，是目前最強大的開源圍棋 AI。

KataGo 的創新點​

KataGo 並非在神經網路架構上做出革命性改變，而是在訓練方法和輔助任務上進行系統性優化，實現了顯著的效率提升。

核心創新總覽​

創新點效果
輔助訓練目標加速學習，提供更多監督信號
全局池化結構更好地捕捉全局資訊
支援多種規則單一模型適應不同比賽規則
Playout Cap 隨機化提高訓練效率
改進的資料增強增加訓練資料多樣性

更高效的訓練方法​

輔助訓練目標（Auxiliary Training Targets）​

傳統的 AlphaGo Zero 只有兩個訓練目標：

  1. Policy：預測 MCTS 的落子機率分佈
  2. Value：預測遊戲勝負

KataGo 添加了多個輔助訓練目標，提供更豐富的學習信號：

各輸出頭說明​

輸出頭維度預測目標
Policy19×19+1每個位置的落子機率（含 Pass）
Value3勝/負/和的機率
Score連續值預測最終目數差
Ownership19×19每個點最終歸屬（黑/白領地）
Short-term Value1短期內的預期勝率
Lead1當前領先多少目

為什麼輔助目標有效？​

  1. 更密集的監督信號：Value 只提供一個數值，而 Ownership 提供 361 個監督點
  2. 減少過擬合：多任務學習有正規化效果
  3. 加速收斂：輔助任務幫助網路更快學習有用的特徵表示
  4. 提供更好的梯度：避免梯度消失問題

Playout Cap 隨機化​

AlphaGo Zero 每手棋固定進行 800 次 MCTS 模擬。KataGo 引入隨機化：

# 傳統方式
num_playouts = 800  # 固定

# KataGo 方式
playout_cap = random.choice([
    100, 200, 300, 400, 500, 600, 700, 800
])

好處：

  • 訓練資料更加多樣化
  • 模型學會在不同搜索深度下做出合理判斷
  • 實際對弈時即使搜索量較少也能表現良好

資料增強改進​

傳統方法利用圍棋的 8 重對稱性（4 次旋轉 × 2 次鏡像）進行資料增強。KataGo 進一步改進：

  • 隨機對稱變換：每次取樣時隨機選擇對稱變換
  • 歷史狀態隨機化：隨機選擇歷史盤面的表示方式
  • 顏色隨機化：隨機交換黑白視角

支援多種圍棋規則​

這是 KataGo 的重要特色之一。不同的圍棋規則會影響戰略決策：

主要規則差異​

規則體系計分方式貼目打劫規則自殺

技術實現​

KataGo 將規則資訊編碼為輸入特徵：

# 規則相關輸入特徵示例
rule_features = {
    'komi': 7.5,           # 貼目值
    'scoring_rule': 'area', # 數子/數目
    'ko_rule': 'simple',    # 劫爭規則
    'suicide_allowed': False,
    'tax_rule': 'none',     # 是否有「眼」稅
    # ...
}

網路學會根據不同規則調整策略。例如：

  • 數目法下更注重實地控制
  • 允許自殺時可用於特殊戰術
  • 不同貼目會影響佈局選擇

同時預測勝率和目數​

這是 KataGo 最實用的功能之一，對於圍棋教學和分析極有價值。

實際應用價值​

  1. 更精確的局面判斷：

    • 勝率 80% 但只領先 1 目 → 還有變數
    • 勝率 80% 且領先 20 目 → 大局已定

  2. 教學輔助：

    • 讓學生理解一手棋「虧了多少目」
    • 比較不同下法的目數差異

  3. 讓子棋分析：

    • 準確評估讓子是否適當
    • 判斷該下進攻還是防守

Score Distribution​

KataGo 不只預測單一目數，而是預測完整的目數分佈：

目數分佈示例：
├─ 黑勝 10 目以上：15%
├─ 黑勝 5-10 目：25%
├─ 黑勝 0-5 目：20%
├─ 白勝 0-5 目：18%
├─ 白勝 5-10 目：15%
└─ 白勝 10 目以上：7%

預期值：黑棋 +3.2 目
標準差：±8.5 目

這種分佈資訊能更好地反映局面的複雜程度和不確定性。

應用場景​

  • 局面分析：一眼看出雙方勢力範圍
  • 死活判斷：判斷某塊棋是否還有救
  • 收官計算：評估各處官子的價值
  • 教學示範：視覺化展示領地概念

訓練效率比較​

達到 ELF OpenGo 水平所需資源：

ELF OpenGo:
- 2000 GPU
- 2 週訓練

KataGo:
- 1 GPU（或幾十 GPU 加速）
- 數天到數週

效率提升：約 50-100 倍

網路架構細節​

全局池化（Global Pooling）​

傳統 CNN 只能看到局部資訊，KataGo 加入全局池化層捕捉全局特徵：

class GlobalPoolingBlock(nn.Module):
    def forward(self, x):
        # x: [batch, channels, 19, 19]

        # 全局平均池化
        global_avg = x.mean(dim=[2, 3])  # [batch, channels]

        # 全局最大池化
        global_max = x.max(dim=2)[0].max(dim=1)[0]  # [batch, channels]

        # 合併全局特徵
        global_features = torch.cat([global_avg, global_max], dim=1)

        # 將全局特徵廣播回空間維度
        global_features = global_features.view(batch, -1, 1, 1)
        global_features = global_features.expand(-1, -1, 19, 19)

        return torch.cat([x, global_features], dim=1)

好處：

  • 能感知全局形勢（如誰在領先）
  • 更好地處理需要全局判斷的局面
  • 對目數預測特別有幫助
