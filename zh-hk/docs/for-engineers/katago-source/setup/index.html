KataGo 安裝同設定

本文將會詳細介紹點樣喺各平台上安裝 KataGo，包括系統需求、安裝步驟、模型選擇同設定檔說明。

系統需求​

硬件需求​

GPU（推薦）​

GPU 類型支援狀態建議後端
NVIDIA（CUDA）最佳支援CUDA
NVIDIA（冇 CUDA）良好支援OpenCL
AMD良好支援OpenCL
Intel 內顯基本支援OpenCL
Apple Silicon良好支援Metal / OpenCL

CPU 模式​

如果冇合適嘅 GPU，可以使用 Eigen 後端純 CPU 運行：

  • 效能較低（約 10-30 playouts/sec）
  • 適合學習、測試同低強度使用
  • 需要 AVX2 指令集支援（大多數現代 CPU 都支援）

軟件需求​

作業系統版本要求
LinuxUbuntu 18.04+, CentOS 7+ 或其他現代發行版
macOS10.14+ (Mojave 或更新)
WindowsWindows 10 或更新

後端選擇指南​

macOS 安裝​

方法 1：Homebrew（推薦）​

最簡單嘅安裝方式，自動處理所有依賴。

# 安裝 KataGo
brew install katago

# 確認安裝成功
katago version

Homebrew 版本預設使用 OpenCL 後端，喺 Intel Mac 同 Apple Silicon 上都可以運行。

方法 2：由原始碼編譯​

適合需要自訂選項或最新功能嘅使用者。

安裝依賴​

# 安裝 CMake 同編譯工具
brew install cmake

# 如果使用 OpenCL
# macOS 已內置 OpenCL 支援，唔使額外安裝

編譯步驟​

# 克隆原始碼
git clone https://github.com/lightvector/KataGo.git
cd KataGo/cpp

# 建立編譯目錄
mkdir build && cd build

# 設定編譯選項（OpenCL 後端）
cmake .. -DUSE_BACKEND=OPENCL

# 編譯
make -j$(sysctl -n hw.ncpu)

# 編譯完成後，執行檔位於 build/katago
./katago version

Apple Silicon 特別說明​

喺 M1/M2/M3 Mac 上，建議使用 OpenCL 或 Metal 後端：

# OpenCL 後端（推薦，相容性較好）
cmake .. -DUSE_BACKEND=OPENCL

# Metal 後端（實驗性，可能有更好效能）
cmake .. -DUSE_BACKEND=METAL

Linux 安裝​

方法 1：預編譯版本（推薦）​

由 GitHub Releases 下載預編譯版本：

# 下載 OpenCL 版本（適用於大多數情況）
wget https://github.com/lightvector/KataGo/releases/download/v1.15.3/katago-v1.15.3-opencl-linux-x64.zip

# 或下載 CUDA 版本（NVIDIA GPU）
wget https://github.com/lightvector/KataGo/releases/download/v1.15.3/katago-v1.15.3-cuda11.1-linux-x64.zip

# 解壓縮
unzip katago-v1.15.3-*.zip

# 賦予執行權限
chmod +x katago

# 確認安裝
./katago version

方法 2：由原始碼編譯​

CUDA 後端（NVIDIA GPU）​

# 安裝依賴
sudo apt update
sudo apt install cmake g++ libzip-dev

# 安裝 CUDA（如果未安裝）
# 請參考 NVIDIA 官方指南：https://developer.nvidia.com/cuda-downloads

# 克隆同編譯
git clone https://github.com/lightvector/KataGo.git
cd KataGo/cpp
mkdir build && cd build

# 設定 CUDA 後端
cmake .. -DUSE_BACKEND=CUDA

# 編譯
make -j$(nproc)

OpenCL 後端（通用 GPU）​

# 安裝依賴
sudo apt update
sudo apt install cmake g++ libzip-dev ocl-icd-opencl-dev

# 安裝 OpenCL 驅動
# NVIDIA: 跟 CUDA 一齊安裝
# AMD: sudo apt install mesa-opencl-icd
# Intel: sudo apt install intel-opencl-icd

# 克隆同編譯
git clone https://github.com/lightvector/KataGo.git
cd KataGo/cpp
mkdir build && cd build

cmake .. -DUSE_BACKEND=OPENCL
make -j$(nproc)

Eigen 後端（純 CPU）​

# 安裝依賴
sudo apt update
sudo apt install cmake g++ libzip-dev libeigen3-dev

# 克隆同編譯
git clone https://github.com/lightvector/KataGo.git
cd KataGo/cpp
mkdir build && cd build

cmake .. -DUSE_BACKEND=EIGEN
make -j$(nproc)

Windows 安裝​

方法 1：預編譯版本（推薦）​

  1. 前往 KataGo Releases
  2. 下載適合嘅版本：
    • katago-v1.15.3-cuda11.1-windows-x64.zip（NVIDIA GPU + CUDA）
    • katago-v1.15.3-opencl-windows-x64.zip（其他 GPU）
    • katago-v1.15.3-eigen-windows-x64.zip（純 CPU）
  3. 解壓縮到指定目錄
  4. 喺命令提示字元入面測試：
cd C:\path\to\katago
katago.exe version

方法 2：由原始碼編譯​

準備環境​

  1. 安裝 Visual Studio 2019/2022（含 C++ 工具）
  2. 安裝 CMake
  3. 如果使用 CUDA，安裝 CUDA Toolkit

編譯步驟​

# 喺 Developer Command Prompt 入面執行

git clone https://github.com/lightvector/KataGo.git
cd KataGo\cpp
mkdir build
cd build

# CUDA 後端
cmake .. -G "Visual Studio 17 2022" -A x64 -DUSE_BACKEND=CUDA

# 或 OpenCL 後端
cmake .. -G "Visual Studio 17 2022" -A x64 -DUSE_BACKEND=OPENCL

# 編譯
cmake --build . --config Release

模型下載同選擇​

KataGo 需要神經網絡模型檔案先可以運行。官方提供多種大細嘅模型。

下載位置​

官方模型下載頁面：https://katagotraining.org/

或直接由訓練伺服器下載：

# 較細模型（適合測試同弱硬件）
curl -L -o kata-b18c384.bin.gz \
  "https://media.katagotraining.org/uploaded/networks/models/kata1/kata1-b18c384nbt-s9996604416-d4316597426.bin.gz"

# 中型模型（推薦，平衡性能同速度）
curl -L -o kata-b40c256.bin.gz \
  "https://media.katagotraining.org/uploaded/networks/models/kata1/kata1-b40c256-s11840935168-d2898845681.bin.gz"

# 大型模型（最強棋力，需要強力 GPU）
curl -L -o kata-b60c320.bin.gz \
  "https://media.katagotraining.org/uploaded/networks/models/kata1/kata1-b60c320-s11318001920-d2792783831.bin.gz"

模型比較​

模型網絡大細檔案大細棋力適用場景
b10c12810 blocks, 128 channels~20 MB業餘高段CPU、弱 GPU、快速測試
b18c38418 blocks, 384 channels~140 MB職業水準一般 GPU、日常分析
b40c25640 blocks, 256 channels~250 MB超人水準中高階 GPU、深度分析
b60c32060 blocks, 320 channels~500 MB頂級超人高階 GPU、頂級分析

選擇建議​

你嘅 GPU 係？
├─ RTX 3080/4080/4090 → b60c320 或 b40c256
├─ RTX 3060/3070 → b40c256 或 b18c384
├─ GTX 1660/2060 → b18c384
├─ 入門級 GPU → b18c384 或 b10c128
└─ 純 CPU → b10c128

基本設定檔說明​

KataGo 使用 .cfg 格式嘅設定檔。官方提供範例設定檔喺 cpp/configs/ 目錄入面。

重要設定檔​

檔案用途
gtp_example.cfgGTP 模式嘅範例設定
analysis_example.cfgAnalysis Engine 嘅範例設定
default_gtp.cfgGTP 模式嘅預設設定

核心設定項目​

# =============
# 神經網絡設定
# =============

# 神經網絡模型檔案路徑
# 可使用相對路徑或絕對路徑
# model = /path/to/model.bin.gz

# =============
# 搜索設定
# =============

# 每手棋嘅最大搜索訪問數
# 增加呢個值會提高棋力但減慢速度
maxVisits = 500

# 每手棋嘅最大思考時間（秒）
# 0 表示無限制（由 maxVisits 控制）
maxTime = 0

# 每次移動使用嘅執行緒數
numSearchThreads = 6

# =============
# 規則設定
# =============

# 圍棋規則
# chinese = 中國規則
# japanese = 日本規則
# korean = 韓國規則
# tromp-taylor = Tromp-Taylor 規則
# aga = AGA 規則
rules = chinese

# 貼目
komi = 7.5

# =============
# GPU 設定
# =============

# 使用嘅 GPU 編號（由 0 開始）
# 多 GPU 可用逗號分隔：0,1,2
# -1 表示自動選擇
nnDeviceIdxs = 0

# 每個 GPU 執行緒嘅批次大細
# 較大嘅批次可提高 GPU 利用率
numNNServerThreadsPerModel = 1

# =============
# 進階設定
# =============

# MCTS 探索常數
# 較大值增加探索，較細值增加利用
cpuctExploration = 1.0

# 根節點嘅 Dirichlet 噪音
# 用嚟增加探索多樣性
rootNoiseEnabled = true
rootDirichletNoisePruneFactor = 0.25

GTP 模式設定範例​

建立 my_gtp_config.cfg：

# 模型路徑
# model 通常喺命令列指定，呢度可以省略

# 搜索設定
maxVisits = 1000
numSearchThreads = 4

# 規則
rules = chinese
komi = 7.5

# GPU 設定
nnDeviceIdxs = 0

# 日誌
logDir = ./logs
logToStderr = false

Analysis Engine 設定範例​

建立 my_analysis_config.cfg：

# 搜索設定
maxVisits = 500
numSearchThreads = 8

# 報告設定
# 每次分析報告嘅最大變化數
maxMoves = 10

# 規則（可喺 API 呼叫嗰陣覆蓋）
rules = chinese
komi = 7.5

# GPU 設定
nnDeviceIdxs = 0
numNNServerThreadsPerModel = 2

# 分析功能
reportAnalysisWinratesAs = BLACK

# 所有權地圖
# 啟用之後可獲得每個點嘅歸屬預測
analysisOwnership = true

首次運行​

測試 GTP 模式​

# 運行 GTP 模式
katago gtp -model /path/to/model.bin.gz -config /path/to/config.cfg

# 如果冇 config，KataGo 會使用預設值
katago gtp -model /path/to/model.bin.gz

成功啟動之後，輸入 GTP 指令測試：

name
= KataGo

version
= 1.15.3

boardsize 19
=

genmove black
= Q16

quit
=

測試 Analysis Engine​

# 運行 Analysis Engine
katago analysis -model /path/to/model.bin.gz -config /path/to/config.cfg

輸入 JSON 查詢：

{"id":"test1","initialStones":[],"moves":[],"rules":"chinese","komi":7.5,"boardXSize":19,"boardYSize":19,"analyzeTurns":[0]}

基準測試​

測試你嘅硬件效能：

# 運行基準測試
katago benchmark -model /path/to/model.bin.gz

# 指定設定檔
katago benchmark -model /path/to/model.bin.gz -config /path/to/config.cfg

基準測試會輸出：

  • 每秒搜索訪問數（visits/sec）
  • 神經網絡推理速度
  • 建議嘅執行緒設定

常見問題排解​

GPU 相關​

問題：搵唔到 GPU

# 檢查 OpenCL 設備
clinfo

# 或列出 KataGo 睇到嘅設備
katago gpuinfo

問題：CUDA 初始化失敗

  • 確認 CUDA 版本同 KataGo 編譯版本相符
  • 更新 GPU 驅動程式
  • 檢查 CUDA_PATH 環境變數

記憶體相關​

問題：記憶體唔夠

# 減少 GPU 記憶體使用
nnMaxBatchSize = 8  # 預設可能係 16 或更高
nnCacheSizePowerOfTwo = 20  # 減少快取大細

效能相關​

問題：速度太慢

  1. 確認使用緊 GPU 而唔係 CPU
  2. 減少 numSearchThreads
  3. 使用較細嘅模型
  4. 檢查係咪有其他程式佔用緊 GPU
